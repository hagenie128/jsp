분실물 보관 시스템 설계 정리

최근 3일 동안 배운 JSP, MVC, MyBatis, AJAX, 동적 HTML 출력, 파일 업로드, 파일 다운로드, 드래그앤드롭을 한 번에 복습하기 위한 예제입니다.

1. 프로젝트 주제

분실물 보관 시스템

학원이나 학교에서 발견된 분실물을 관리자가 등록하고, 사용자는 분실물 목록을 검색할 수 있는 시스템입니다. 분실물에는 물건명, 발견장소, 보관상태, 사진 파일이 포함됩니다.

핵심 기능

  • 분실물 전체 목록 조회
  • 분실물 검색: 물건명, 발견장소, 보관상태 기준
  • AJAX 검색 후 tbody 동적 출력
  • 분실물 등록
  • 분실물 수정
  • 분실물 삭제
  • 분실물 사진 업로드
  • 드래그앤드롭 파일 업로드
  • 첨부파일 다운로드

2. DB 테이블 설계

테이블명: lost_item

컬럼명 자료형 설명
item_no int primary key auto_increment 분실물 번호
item_name varchar(100) 분실물 이름
found_place varchar(100) 발견 장소
found_date date 발견 날짜
item_status varchar(20) 보관중, 반환완료, 폐기예정
description varchar(500) 상세 설명
origin_file_name varchar(255) 사용자가 업로드한 원본 파일명
save_file_name varchar(255) 서버에 저장된 파일명
file_path varchar(500) 파일 저장 경로
created_at datetime 등록일

SQL 생성문

			
			create database lost_item_db;
			use lost_item_db;

create table lost_item (
	item_no int primary key auto_increment,
	item_name varchar(100) not null,
	found_place varchar(100) not null,
	found_date date not null,
	item_status varchar(20) default '보관중',
	description varchar(500),
	origin_file_name varchar(255),
	save_file_name varchar(255),
	file_path varchar(500),
	created_at datetime default now());

6. MyBatis SQL 구성

lost_item_mapper.xml에 넣을 SQL 방향

전체 조회

			select *
from lost_item
order by item_no desc

검색 조회

			<choose>
	<when test="kind == 'itemName'">
		item_name like concat('%', #{search}, '%')
	</when>
	<when test="kind == 'foundPlace'">
		found_place like concat('%', #{search}, '%')
	</when>
	<when test="kind == 'itemStatus'">
		item_status = #{search}
	</when>
</choose>

등록

			insert into lost_item
(item_name, found_place, found_date, item_status, description, origin_file_name, save_file_name, file_path)
values
(#{itemName}, #{foundPlace}, #{foundDate}, #{itemStatus}, #{description}, #{originFileName}, #{saveFileName}, #{filePath})

set item_name = #{itemName},
	found_place = #{foundPlace},

10. AJAX 검색 작동 흐름

			1. 사용자가 검색 기준과 검색어를 입력한다.
2. 검색 버튼을 누른다.
3. JS가 kind와 search 값을 가져온다.
4. fetch('./searchLostItem.do?kind=' + kind + '&search=' + search) 요청을 보낸다.
5. Controller가 kind와 search를 받는다.
6. Service가 Mapper를 호출한다.
7. Mapper가 DB에서 검색 결과를 가져온다.
8. Controller가 JSON 형태로 응답한다.
9. JS가 data.list를 받는다.
10. tbody id="listArea"를 비운다.
11. forEach로 tr을 조립한다.
12. area.innerHTML = tag로 화면을 다시 그린다.

검색 JSON 응답 형태

			{
	"result": 1,
	"msg": "3건 조회",
	"list": [
		{
			"itemNo": 1,
			"itemName": "검은색 지갑",
			"foundPlace": "3층 강의실",
			"foundDate": "2026-05-13",
			"itemStatus": "보관중",
			"description": "학생증과 카드가 들어있는 검은색 지갑",
			"originFileName": "wallet.jpg"
		}
	]
}

11. 파일 업로드 작동 흐름

			1. 사용자가 분실물 정보를 입력한다.
2. 사용자가 사진 파일을 선택한다.
3. 업로드 버튼을 누른다.
4. JS가 FormData를 만든다.
5. 일반 input 값은 input.value로 꺼내서 formData.append 한다.
6. 파일 input 값은 input.files[0]으로 꺼내서 formData.append 한다.
7. fetch('./fileUpload.do', { method: 'POST', body: formData })로 전송한다.
8. Controller는 @MultipartConfig 상태에서 request.getParts()로 데이터를 받는다.
9. Part의 getSubmittedFileName()이 있으면 파일로 판단한다.
10. File root = new File('C:/fileupload')로 저장 폴더를 지정한다.
11. 폴더가 없으면 mkdirs()로 만든다.
12. part.write(savePath)로 파일을 저장한다.
13. 파일명과 경로를 DTO에 담아 DB에 insert 한다.
14. JSON으로 업로드 성공 메시지를 응답한다.

12. 드래그앤드롭 작동 흐름

이벤트명 역할
dragenter 파일이 업로드 영역 안으로 들어올 때 스타일 변경
dragover 파일이 영역 위에 있을 때 발생. event.preventDefault() 필수
dragleave 파일이 영역 밖으로 나갈 때 스타일 원래대로 변경
drop 파일을 놓았을 때 dataTransfer.files로 파일 목록을 받음 
			1. 사용자가 파일을 dropArea 위로 끌고 온다.
2. dragover에서 event.preventDefault()를 호출한다.
3. drop 이벤트에서 event.dataTransfer.files를 꺼낸다.
4. selectedFiles 배열에 파일을 저장한다.
5. fileList 영역에 파일명을 출력한다.
6. 업로드 버튼을 누르면 selectedFiles를 FormData에 담는다.
7. fetch로 서버에 전송한다.

13. 파일 다운로드 작동 흐름

			1. 사용자가 다운로드 버튼을 누른다.
2. location.href = './fileDownload.do?itemNo=1'로 요청한다.
3. Controller가 itemNo를 받는다.
4. DB에서 파일 경로와 원본 파일명을 조회한다.
5. File 객체로 실제 파일을 잡는다.
6. response.setHeader('Content-Disposition', 'attachment; filename="파일명"')을 설정한다.
7. FileInputStream으로 서버 파일을 읽는다.
8. response.getOutputStream()으로 브라우저에 전송한다.
9. 브라우저가 파일을 다운로드한다.
주의: Content-Disposition 철자가 틀리면 다운로드가 아니라 브라우저에 내용이 출력될 수 있습니다.
